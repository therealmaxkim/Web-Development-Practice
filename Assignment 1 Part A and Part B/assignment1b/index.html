Merge K Sorted Lists

Problem Description

Merge k sorted linked lists and return it as one sorted list. Analyze and describe its complexity.

Examples

Input:

[

1->4->5
1->3->4
2->6

]

Output: 1->1->2->3->4->4->5->6

Questions

  1. can you modify the linked lists from input?
  2. can the linked lists be empty?

Observations

  1. The length of each list will not always be the same
  2. all lists are in sorted order
  3. this problem is similar to "merge 2 sorted lists", but now extend to n lists

Ideas

Idea 1: brute force

What if you have pointers for each linked list, and you have a new linked list and just add the one with the lowest value. Along the way, one pointer/linked list will end. In that case, you just ignore that one.

Keep a list of pointers and iterate through each one and compare if the pointer is not None. This will run in O(l*n), where n is the total number of nodes with O(l) space, where l is the total number of linked lists. (constant space if we can modify input.)

How do you know when you are done? when all linked lists are None.

If you can modify the original list, in order to increment the node with the lowest value, you must save the index which the linked list is on

Idea 2: using a heap

in order to find the smallest element in that array, it takes O(l) time each run. So how do you reduce the time needed to find the smallest element? use a min heap. But, you will have to also put that array's next element after you take out the smallest element, similar to how you would with merge 2 sorted arrays.

For each linked list, append the values to a min heap. Pop the lowest value from the min heap in constant time. Then increment the pointer which that value came from. Keep going until min heap is empty
