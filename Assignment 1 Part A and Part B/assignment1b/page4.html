Set Matrix Zeroes

Problem Description

Given a m x n matrix, if an element is 0, set its entire row and column to 0. Do it in-place.

Examples

Input:

[

[1,1,1],
[1,0,1],
[1,1,1]

]

Output:

[

[1,0,1],
[0,0,0],
[1,0,1]

]

Input:

[

[0,1,2,0],
[3,4,5,2],
[1,3,1,5]

]

Output:

[

[0,0,0,0],
[0,4,5,0],
[0,3,1,0]

]

Follow up:

  • A straight forward solution using O(mn) space is probably a bad idea.
  • A simple improvement uses O(m + n) space, but still not the best solution.
  • Could you devise a constant space solution?

Questions

  1. Can the matrix be empty?

Observations

  1. Once you see zero and set its entire row and column to 0, you want to avoid redoing that work if there exists another zero in the same row/column.
  2. You need to "remember" which rows/columns were set to zero so that you can skip those rows/columns.
  3. This must be done in place in O(1) space.
  4. You must do changes after, because if you change the matrix while searching for zeros, you cannot differentiate between zeros that existed in the matrix versus zeros that you set.
  5. If you make changes along the way, you cannot differentiate between zeros that were in the matrix versus zeros that were set

Ideas

Idea 1: O(n+m) solution

  1. remember which row/columns need to be set to zero. Only change the matrix at the very end.
  2. keep a set of each row and column numbers not visited
  3. run through each pair of row and columns from the two sets, and check if zero
  4. if zero, then set zeros and remove that zero's row and column from the two sets

Problem with solution: updates to the set. sets dont keep order. hard to avoid repeats because unordered and some numbers can be deleted midway.

Idea 2: constant space

  1. for each zero, set itself as "None"
  2. go through the matrix again and set each row&column of "None" as 0
  3. if you come across another "None" while setting Nones as zero, ignore it.
  4. this is necessary to be able to differentiate between zeros that are set and zeros in the matrix.

Correct idea: efficient constant space

if you come across a zero, set the first element in the row/col to zero.

This is needed because the brute force constant space can revisited already set cells. Doing it this way will remove any confusion about a zero being set or already existing in the original matrix.

edge case: for the first row and column, you would not be able to know. So keep two variables to tell if the first row or column have been set.
